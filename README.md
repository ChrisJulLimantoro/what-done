## Install

```bash
npm install -g whatdone
```

Requires Node.js 18+.

---

## Setup

Run the setup wizard once after installing:

```bash
whatdone config init
```

The wizard will:
1. Ask which LLM provider to use (Anthropic, Gemini, or OpenAI)
2. Help you get and save your API key to `~/.zshrc` (or `~/.bashrc`)
3. Configure git repo discovery
4. Detect your Claude Code session path

Then activate your API key:

```bash
source ~/.zshrc   # or open a new terminal tab
```

---

## API Keys

whatdone reads your API key from environment variables. Set one of:

| Provider  | Environment Variable  | Get Key |
|-----------|-----------------------|---------|
| Anthropic | `ANTHROPIC_API_KEY`   | [console.anthropic.com](https://console.anthropic.com/settings/keys) |
| Google Gemini | `GEMINI_API_KEY`  | [aistudio.google.com](https://aistudio.google.com/app/apikey) |
| OpenAI    | `OPENAI_API_KEY`      | [platform.openai.com](https://platform.openai.com/api-keys) |

Provider is auto-detected from whichever key is set. If multiple are set, Anthropic takes priority.

## Commands

### `whatdone today`
Summarize today's git commits and Claude Code sessions.

```bash
whatdone today
whatdone today --dry-run        # preview the prompt without calling the LLM
whatdone today --no-cache       # force regeneration even if cached
whatdone today --date 2026-04-24  # summarize a specific date
whatdone today --provider gemini  # override LLM provider
```

### `whatdone standup`
Format today's summary as a standup update (Yesterday / Today / Blockers).

```bash
whatdone standup
whatdone standup --date 2026-04-24
```

### `whatdone weekly`
Summarize the past 7 days. Reads from saved daily snapshots. If any days in the range are missing a snapshot, you'll be prompted to generate them on the spot before the summary runs.

```bash
whatdone weekly
whatdone weekly --since 2026-04-18   # custom start date
whatdone weekly --provider openai    # override LLM provider
```

### `whatdone config`
Show current configuration and detected LLM provider.

```bash
whatdone config
whatdone config init   # re-run setup wizard
```

## Configuration

Config is stored at `~/.whatdone/config.toml`. Generated by `whatdone config init` вЂ” you rarely need to edit it manually.

```toml
[llm]
provider = "anthropic"
max_tokens_per_chunk = 3000
max_chunks = 10
max_summary_tokens = 2000

[git]
auto_discover = true      # scan ~/ for all git repos automatically
repos = []                # used when auto_discover = false
max_diff_tokens = 4000

[claude_code]
session_path = "~/.claude/projects"
enabled = true

[output]
format = "text"

[template]
grouping = "flat"   # "flat" or "by_project"
```

## Customizing the Summary

You can fully control what sections appear in your summary and how they are grouped.

### Sections

Each section has a `name`, `type` (`string` or `list`), and a `description` that is passed to the LLM as an instruction. Add, remove, or rename sections by editing `~/.whatdone/config.toml`:

```toml
[template]
grouping = "flat"

[[template.sections]]
name = "oneLiner"
type = "string"
description = "One sentence summary of the day"
required = true

[[template.sections]]
name = "narratives"
type = "list"
description = "2-4 bullet points describing major work streams"

[[template.sections]]
name = "bullets"
type = "list"
description = "6-10 granular bullet points of specific changes"

[[template.sections]]
name = "themes"
type = "list"
description = "2-4 theme keywords like 'refactoring', 'bug fixes', 'new feature'"
```

**Example вЂ” add a "PR-ready summary" section:**

```toml
[[template.sections]]
name = "prSummary"
type = "string"
description = "A one-paragraph summary suitable for a pull request description"
```

**Example вЂ” minimal setup:**

```toml
[[template.sections]]
name = "summary"
type = "string"
description = "A brief summary of today's work in 2-3 sentences"
required = true

[[template.sections]]
name = "highlights"
type = "list"
description = "3-5 key things accomplished"
```

> Changing sections invalidates your local cache so the next run regenerates with the new schema.

### Grouping by project

If you work across multiple git repos each day, set `grouping = "by_project"` to get a separate summary per repo:

```toml
[template]
grouping = "by_project"
```

Output will show one box per project, with each project's sections rendered inside it. A merged flat summary is still saved to the snapshot for use by `whatdone standup` and `whatdone weekly`.

## Structured Outputs

All three LLM providers use native structured-output features вЂ” no fragile JSON parsing:

- **Anthropic**: tool_use with forced `emit_summary` tool
- **OpenAI**: `response_format: json_schema`
- **Gemini**: `responseMimeType: application/json` + `responseSchema`

This eliminates the raw-JSON rendering issue and guarantees the summary always matches your configured sections.

## Privacy

- Code diffs are sent to your chosen LLM provider only вЂ” nowhere else
- Files matching sensitive patterns are redacted before any LLM call: `.env`, `*secret*`, `*credential*`, `*password*`, `*.key`, `*.pem`
- Summaries are cached locally at `~/.whatdone/cache/` for 1 hour
- Daily snapshots are saved at `~/.whatdone/snapshots/YYYY-MM-DD/`

Use `--dry-run` to inspect exactly what gets sent to the LLM before running for real.
